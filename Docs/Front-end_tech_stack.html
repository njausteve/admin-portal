Proposed new component

1. UI-Router

Version: 1.0.0
Link : https://ui-router.github.io
License: The MIT License

Description:
Angular UI-Router is a client-side Single Page Application routing framework for AngularJS.
Routing frameworks for SPAs update the browser’s URL as the user navigates through the app. Conversely, this allows changes to the browser’s URL to drive navigation through the app, thus allowing the user to create a bookmark to a location deep within the SPA.

UI-Router applications are modeled as a hierarchical tree of states. UI-Router provides a state machine to manage the transitions between those application states in a transaction-like manner.

Specific Use-Case in application:
To achieve nested routing.

Possiblity with current reFrame code :
Implementing nested routing is not possible with current implementation of reFrame with simple Angular router.

2. OcLazyLoad

Version: 1.1.0
Link : https://github.com/ocombe/ocLazyLoad
License: The MIT License

Description:
OcLazyLoad is used to load the dependency files and also to inject the AngularJS modules on the fly. Perfectly suitable to load JS or even CSS async. A recommended lazy loader for Angular 1.X .

Key features:
- Dependencies are automatically loaded Debugger friendly
- The ability to mix normal boot and load on demand Load via the service or the directive
- Use the embedded async loader or use your own (Web Pack, requireJS, …)
- Load js / css / templates files
- Compatible with AngularJS 1.2.x/1.3.x/1.4.x/1.5.x/1.6.x

Specific Use-Case in application:
To achieve Lazy-loading and dependency injection on the fly.

Possiblity with current reFrame code :
Currently this feature is impelemted using RequireJS, which is little heavy for mobile websited like AppMart. As well have following disadvantages considering mobile website:

  • RequireJS is mainly used to load the dependency js files.
  • We can’t inject the AngularJS modules on the fly. (A dependency Injection on the fly.)
  • Minification and concatination is little difficult of require files using Gulp uglifier or minifier.

3. Service Worker

A Browser feature

Description:
Service Worker is a brand new Cache API provided by Modern Browsers. A major advantage of the service worker cache API is that it gives you more detailed control than the built-in browser cache does. Older browsers will work with a fall-back to normal browser cache mechanism.

Specific Use-Case in application:
To achieve fine tune control over browser cache.
